2017-11-01 至今.

博客又重新开了.

最开始确实很有很多文章, 不过到后来内容丢了, 原来的实战文章也因为某些原因不能发出去.

以前写过工具推荐的, 比如什么 awvs burpsuite sqlmap 都给你单独开一个文章凑数.

到后来转向各种已公布的 cms 中间件 服务器漏洞, 随便写一下漏洞说明, 给出 payload 就完事了, 当然也是凑字数.

文章改来改去, 删来删去, 到现在就变成了这样.

不敢说我写的内容多么牛逼, 但至少这些都是我在实战中实际遇到过的情况, 然后逐一写成文章记录下来.

其实我是个很不自信的人.

一步一步搞下来的站, 在自己看来其实过程都蛮简单的.

"要是当时再细心一点或许就会更容易了吧?"

过程要不就是行云流水的那种, 或者是最后发现了某个一开始就没关注到的突破点.

我是个实战派, 我讨厌某些为出题而出题的 ctf 题目, 讨厌某些为排名而刷 src 的白帽子.

渗透的本质是什么? 是渗透.

我不管你用多华丽的手法绕过了这个sqli xss ssrf, 但你只要没有日进去, 你就是垃圾.

我不管你用多效率的工具刷出了目标的一堆信息泄露 csrf jsonp hijacking, 但你只要没有拿到目标站的权限, 你就是废物.

我也不是说这些漏洞就根本没有用, 但这些漏洞在真正的实战中可能没有丝毫用处.

曾经看过一道关于 misc 的 ctf 题目, 用文言文来加密 flag, 密文对照表(类似 ASCII 码表)竟在出题人在百度文库上传的某篇 doc 里.

某些白帽子, 利用 google 爬出一堆的 sqli, 但这些漏洞根本获取不到有用的数据, 因为他 and 1=1 正确 and 1=2 错误所以他就是个漏洞, 然后提交到 src 上.

更有些白帽子, 使用各种所谓的“挖洞神器”, 各种批量自动化刷洞提交, 毫无意义, 仅仅增长了"白帽子"们的浮躁 虚荣之心.

甚至有些所谓的 ctf 大佬连渗透都不会.

我也讨厌i春秋论坛, i春秋的课程蛮好的, 但论坛质量却低的不能再低.

论坛中当然也有思路很精彩的文章 奇淫技巧, gh0stkey jasonx 等版主也在为论坛做贡献.

但用户水平参差不齐, 例如:

渗透刷洞简单思路和解决方式

记一次以小勃大紧张刺激的渗透测试

打造自己的渗透测试环境

记一次突破网站后台

听说有小伙伴想知道暗网怎么去

手把手教你如何用py编写C段批量扫描器

一次对av站点的轻松渗透

标题党就姑且不说.

注入上传一条龙 getshell 的文章还用得着发出来?

python 脚本就一个 requests 然后正则或者其它方式提取内容, 顶多加上个多线程, 还用得着你手把手教?

zoomeye shodan api 获取网站 exp 逐个打就是挖洞了? 就是渗透了?

打造自己的渗透环境? 看标题觉得很厉害, 但无非就是给你先装上 jdk python ruby perl, 装几个编辑器最后再到 github 上 pull 几个开源工具. 哦对了, 还有所谓的 kali 和 pentestbox.

如何去暗网这种百度上一堆的文章无非就是水贴的, 就好比专门发帖教大家如何使用百度搜索.

最后留下自己的 qq 并注明"求好友一起交流".

我 2017 年就放弃了 i春秋论坛, 现在已经 2019 年了, 这种帖子的数量依旧没有下降的趋势.

我注重实战, 你所掌握的原理必须能在实战中体现出来.

我不讨厌新手, 但我讨厌那些给新手指上歪路的人.

所以那些张口就来 “先学一遍 html css javascript sql cmd shell” 的人给我滚远点.

新手入门没成功, 倒是先成前端, DBA, 运维了.

我也讨厌那些"明明不懂技术却装作自己很懂"的人.
